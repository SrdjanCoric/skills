# Feature-pivot handoff template

The next agent must **start a newly-identified piece of work** — a new feature or direction that
surfaced during the current work, spun off for a fresh agent to explore or build. This is
forward-looking; it is *not* parking-and-resuming existing work. Keep it terse and
`path:line`-precise.

## Objective
The new feature/direction to explore or build (from the argument).

## What sparked it
The insight or moment in the *current* work that led here. **Load-bearing** — the new agent has
none of this conversation, so without the "why does this exist" the idea arrives contextless.

## What we've figured out so far
The shape of the idea: scope, constraints, and any decisions already reached in conversation — so
they're not relitigated.

## How it relates to existing code
What it builds on, touches, or replaces. `path:line` + a one-line role for each.

## Open questions
What still needs deciding before or during the build.

## Suggested starting point
Where the fresh agent begins — usually *not* "write code" but "flesh it out."

## Gotchas / constraints
Anything that would bite a build.

## Suggested skill
`talk-it-through`.
